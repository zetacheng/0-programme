# Result Status Policy

## Dependency statuses

`PROPOSED`, `UNVERIFIED`, `ACCEPTED`, `SUSPENDED`, `FAILED`, and `SUPERSEDED` apply to cross-paper dependencies.

## Programme gate statuses

`PROPOSED`, `SPECIFIED`, `OPEN`, `RUNNING`, `PASS`, `FAIL`, `INCONCLUSIVE`, `SUSPENDED`, and `RETIRED` apply to programme gates.

## Global claim statuses

`PROPOSED`, `SUPPORTED`, `VERIFIED`, `CONDITIONAL`, `FAILED`, `RETIRED`, and `INCONCLUSIVE` apply to programme claims.

## Promotion rule

Programme-level `VERIFIED` requires an accepted upstream gate, reproducible evidence, independent review, and synchronization into every affected paper repository. A result is not verified merely because it appears in conversation history or a programme summary.

Statuses must preserve the distinction between the original four-dimensional model, a model extension, a phenomenological EFT, a numerical proxy, and ontology or outlook.
